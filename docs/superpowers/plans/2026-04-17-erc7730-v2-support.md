# ERC-7730 v2 Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ERC-7730 v2 spec compliance alongside existing v1, with a version toggle so users can choose. Changes are modular: types, generation, validation/forms, and UI live in separate files.

**Architecture:** A new `SchemaVersion` type (`"v1" | "v2"`) flows through the store. V2 adds: correct format keys with param names, `interpolatedIntent`, `visible` field property, missing field formats (`chainId`, `tokenTicker`, `enum` enabled), `calldata` params form, `metadata.constants`/`enums`/`token`/`contractName` editing, `$schema` v2 URL, and drops deprecated `contract.abi`. The existing v1 code stays untouched; v2 logic is additive.

**Tech Stack:** Next.js 15, React 18, TypeScript, Zustand, React Hook Form, Zod, Radix UI/shadcn

### Task 2: Fix format key generation to include parameter names (v2)

**Files:**

- [ ] **Step 1: Add v2-aware `computeSignatureWithNames` function**

Add a new function below `canonicalType` in `generate-descriptor.ts`:

```typescript
function namedSignature(name: string, inputs: ABIInput[]): string {
  const params = inputs
    .map((i) => `${canonicalType(i)} ${i.name}`)
    .join(",");
  return `${name}(${params})`;
}
```

- [ ] **Step 2: Update `generateDescriptor` to accept schemaVersion and use named keys for v2**

Change the function signature and body:

```typescript
export function generateDescriptor(params: {
```

Change the format key computation:
